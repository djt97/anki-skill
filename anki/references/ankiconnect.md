## Connection and capability checks

Before any collection operation, run the bundled client from the skill root:

```bash
python3 scripts/anki_client.py version
```

The client uses `http://127.0.0.1:8765`, AnkiConnect API version 6, a bounded timeout, JSON validation, and Python's standard library only. The environment variable `ANKI_CONNECT_URL` may override the URL when the user has deliberately configured a different local endpoint.

If the version call fails, tell the user:

> Anki doesn't seem to be running, or AnkiConnect isn't installed. Please open Anki and try again.

Then stop. Do not install software or change AnkiConnect's configuration.

Before using an unfamiliar action, check it rather than inventing it:

```bash
python3 scripts/anki_client.py apiReflect \
  '{"scopes":["actions"],"actions":["cardsInfo","updateNoteFields"]}'
```

A missing action is unavailable. Do not substitute an undocumented action with a similar-sounding name.

For a one-off action from Python, import the bundled helper:

```python
from scripts.anki_client import call

card_ids = call("findCards", {"query": "is:suspended"})
```

When a script is executed from `scripts/`, this import form also works:

```python
from anki_client import call
```

## Read-only calls

### Search

Use Anki's own search syntax. Quote values containing spaces or punctuation.

```python
call("findCards", {"query": "is:suspended"})
call("findCards", {"query": "tag:leech"})
call("findCards", {"query": "-tag:marked"})
call("findCards", {"query": 'deck:"My Deck"'})
call("findCards", {"query": 'note:"Basic (and reversed card)"'})
call("findCards", {"query": '"mitochondria"'})
call("findCards", {"query": "added:7"})
call("findCards", {"query": "rated:3"})
call("findCards", {"query": "prop:lapses>=8"})
call("findCards", {"query": "prop:ivl<7"})
call("findCards", {"query": 'is:suspended tag:leech deck:"Biology"'})
```

Do not treat `tag:leech` and `prop:lapses>=8` as interchangeable. The first reflects the collection's tags; the second is a search over lapse count.

### Read card content and review signals

Use bounded batches for large ID lists:

```python
from scripts.anki_client import call, cards_info, notes_info

card_ids = call("findCards", {"query": "is:suspended"})
cards = cards_info(card_ids)

note_ids = sorted({int(card["note"]) for card in cards})
notes = notes_info(note_ids)
tags_by_note = {int(note["noteId"]): note["tags"] for note in notes}
```

`cardsInfo` returns card-level data including:

- `cardId`: card ID
- `note`: note ID as an integer, not a nested object
- `fields`: all note fields
- `modelName` and `deckName`
- `factor`, `interval`, `reps`, and `lapses`

Use `notesInfo` for tags and the list of all cards generated by a note.

### Display safely

Create a plain-text copy only for display and analysis. Never write stripped text back to Anki. Before an edit, re-fetch the complete raw field value.

Preserve these elements unless the approved proposal explicitly changes them:

- HTML structure and entities
- `<img>` and other media references
- `[sound:filename]` markers
- MathJax or LaTeX delimiters
- cloze markers and numbering such as `{{c1::answer::hint}}`
- field content not shown in a truncated report

## Card IDs and note IDs

A **note** contains fields and tags. A **card** is a review item generated from a note. One note may generate multiple cards.

| Operation | Identifier |
| --- | --- |
| `findCards`, `cardsInfo` | card IDs |
| `suspend`, `unsuspend`, `changeDeck` | card IDs |
| `updateNoteFields`, `deleteNotes` | note IDs |
| `addTags`, `removeTags`, `notesInfo` | note IDs |

Convert card to note:

```python
card = call("cardsInfo", {"cards": [card_id]})[0]
note_id = int(card["note"])
```

Before changing a note field, call `notesInfo` and preview every card in its `cards` list. A field edit may change multiple sibling cards even when the user selected only one card.

Before deleting a note, show every generated card. `deleteNotes` removes the note and all of its cards.

## Approved mutations

Every example in this section assumes the exact operation has already been previewed and explicitly approved.

### Edit fields

Only include fields that the user approved:

```python
call("updateNoteFields", {
    "note": {
        "id": note_id,
        "fields": {
            "Front": "Approved question",
            "Back": "Approved answer",
        },
    }
})
```

After the call, re-read the note and compare the stored fields with the approved preview.

For cloze notes, preserve all unrelated cloze numbers. Do not merge, add, delete, or renumber clozes unless the preview made that effect explicit.

### Tags

```python
call("addTags", {"notes": [note_id_1, note_id_2], "tags": "reviewed rewritten"})
call("removeTags", {"notes": [note_id], "tags": "leech"})
```

Do not invent a change-tracking tag. Use only a tag the user has confirmed, or ask whether to skip tagging.

### Suspend or unsuspend

These actions take card IDs:

```python
call("suspend", {"cards": [card_id]})
call("unsuspend", {"cards": [card_id_1, card_id_2]})
```

Suspension is allowed by this skill but still requires an exact preview and approval.

### Delete notes

Deletion is permanent from the collection's perspective and affects every card generated by each note:

```python
call("deleteNotes", {"notes": [note_id]})
```

Require a separate confirmation immediately before this call. Never combine deletion with unrelated approved operations.

### Move cards between decks

`changeDeck` takes card IDs. Moving one sibling card does not necessarily move every card generated by the note.

```python
call("changeDeck", {"cards": [card_id], "deck": "New Deck"})
```

Preview the source deck, destination deck, exact card IDs, and sibling-card layout.

### Create notes

```python
call("addNote", {
    "note": {
        "deckName": "My Deck",
        "modelName": "Basic",
        "fields": {"Front": "Q", "Back": "A"},
        "tags": ["my-tag"],
    }
})
```

Confirm the deck, model, every field, and every tag. Check for duplicate or empty-note errors and re-read the created note.

## Scheduling operations that are forbidden

Do not call or emulate operations that alter scheduling or review state, including:

- `answerCards`
- `forgetCards`
- `relearnCards`
- `setDueDate`
- `setEaseFactors`
- direct card-field mutation such as changing `ivl`, `due`, `factor`, `queue`, `reps`, or `lapses`
- deck-option changes that affect scheduling or FSRS
- rescheduling filtered decks
- importing or exporting scheduling data as part of a fix workflow

Do not use `multi` to hide or indirectly perform a forbidden action.

## Batch behavior

AnkiConnect's `multi` action batches requests, but it is **not a transaction**. Earlier sub-actions may succeed even if a later one fails.

```python
result = call("multi", {
    "actions": [
        {"action": "unsuspend", "params": {"cards": [id1, id2]}},
        {"action": "addTags", "params": {"notes": [nid1, nid2], "tags": "reviewed"}},
    ]
})
```

Use it only when every sub-action was approved. Inspect every returned `result`/`error` envelope, report partial failure clearly, and verify final state by re-reading affected objects. For destructive or source-file operations, prefer small sequential calls with verification.

## Filtered decks

Do not call `createFilteredDeck`: it is not a supported AnkiConnect action.

To preserve the filtered-deck outcome:

1. Build and show the exact Anki search string.
2. Confirm that it excludes suspended, buried, or already-filtered cards when relevant, because filtered decks cannot pull those cards in.
3. Tell the user to choose **Tools → Create Filtered Deck** in Anki and paste the search.
4. Require **rescheduling disabled** to comply with this skill's scheduling rule.
5. Do not claim the deck was created or populated unless the user confirms it in Anki.

A read-only alternative is to open the Browser at the search with `guiBrowse`, but only do that if the user wants Anki's UI changed and has approved it.

## Externally authored cards

Only run an external-source check when `external_source` is saved in the user's conventions or the user mentions Obsidian or another external authoring workflow.

A note is externally managed when its note ID appears in the configured source tree. For obsidian-to-anki style markers:

```bash
NOTE_ID='1712345678901'
SOURCE_DIR='/absolute/path/to/source'
grep -RIl --fixed-strings "<!--ID: ${NOTE_ID}-->" "$SOURCE_DIR"
```

- A match means the external file is the source of truth. Show the exact file diff, obtain approval, edit only that file, verify the diff, and remind the user to re-run their sync.
- No match means the note may be Anki-native. Confirm the result before using `updateNoteFields` when external authoring is otherwise expected.

Do not scan arbitrary directories. Use only the source directory the user supplied or previously confirmed.

## HTML, media, and cloze safety

- Strip HTML only in a temporary display representation.
- Preserve raw field values until an approved edit is applied.
- Do not lose image tags, audio markers, MathJax, or hidden context fields.
- Check every cloze marker before and after an edit.
- A `{{c1::...}}` and a `{{c2::...}}` in one note create separate cards; do not renumber them accidentally.
- If a proposed rewrite changes a shared field, preview the rendered meaning of every sibling card.

## Large collections and error handling

- Use targeted searches whenever possible.
- Fetch `cardsInfo` and `notesInfo` in batches of about 200; the bundled helper defaults to 200.
- Use `even_sample` rather than the first N IDs for representative analysis.
- Do not silently drop missing-card placeholders, API errors, or partial `multi` failures.
- Do not retry a mutation automatically after an ambiguous network failure. Re-read state first to determine whether it succeeded.
- For read-only calls, a bounded retry may be reasonable after a transient failure, but report persistent failure.
